## Objective

Wire notification generation into session event processing. When approval requests arrive, sessions fail, or sessions complete, automatically create notification records. Push delivery is stubbed (log only) — actual FCM/APNs integration deferred until credentials are available.

## Acceptance Criteria

- [ ] ApprovalRequest event → creates urgent notification "Session needs your input"
- [ ] Session state → failed → creates high priority notification "Session failed"
- [ ] Session state → completed → creates normal priority notification "Session completed"
- [ ] Notification body includes session title and machine name
- [ ] Notification deep_link points to /sessions/{id}
- [ ] Duplicate prevention: don't re-notify for the same session+type if unread notification exists
- [ ] Push delivery stub: tracing::info log when notification would be pushed
- [ ] `cargo build` and `cargo test` pass
