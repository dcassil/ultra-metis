# Cross-Platform Installer Build and Auto-Update

## Parent Initiative

[[SMET-I-0098]] — Installable Machine Runner with System Tray UI

## Objective

Configure the Tauri bundler to produce installable packages for all three platforms, set up the auto-update mechanism via Tauri's updater plugin, and create the GitHub Actions CI pipeline for release builds.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `tauri.conf.json` configured with app metadata: name "Cadre Machine Runner", identifier `com.cadre.machine-runner`, version from workspace
- [ ] macOS: `cargo tauri build` produces a `.dmg` installer with drag-to-Applications layout
- [ ] Windows: `cargo tauri build` produces an `.msi` installer via WiX
- [ ] Linux: `cargo tauri build` produces `.deb` and `.AppImage` packages
- [ ] App icon set for all platforms (can be a placeholder Cadre logo)
- [ ] Tauri updater plugin configured: checks GitHub Releases for updates on startup (if `auto_update` enabled)
- [ ] Update check endpoint: `https://github.com/{owner}/{repo}/releases/latest/download/latest.json` (Tauri's standard format)
- [ ] "Check for Updates" Tauri IPC command that triggers a manual update check
- [ ] GitHub Actions workflow: on tag push (`v*`), builds release binaries for macOS (x64+arm64), Windows (x64), Linux (x64), uploads to GitHub Release
- [ ] Workflow produces the `latest.json` manifest that the Tauri updater reads
- [ ] `Makefile` updated with `make build-desktop` target

## Implementation Notes

### Technical Approach
- Tauri bundler handles most of the work — the main config is in `tauri.conf.json`
- Code signing: macOS requires Apple Developer cert for notarization (skip in MVP, add as follow-up). Windows Authenticode optional.
- GitHub Actions matrix build: `macos-latest`, `windows-latest`, `ubuntu-latest` runners
- Tauri updater uses GitHub Releases as the update source — no custom server needed
- The `latest.json` file is auto-generated by the Tauri GitHub Action (`tauri-apps/tauri-action`)

### Dependencies
- SMET-T-0265 (Tauri scaffold — provides the app to build)
- All other tasks should be functionally complete before release builds
